# V+D Visual Form Automation — Client Design Preference Collection

> **One-line hook**: Capturing a client's aesthetic preferences in architecture and interior design is inherently visual — but V+D's intake process relied on text forms. I built a visually-driven survey automation that transformed client onboarding into an image-led experience, reducing manual workload by ~50% in early project stages.

## Problem

V+D's pre-engagement process for collecting client design preferences relied on text-based forms — a poor match for clients who think visually (as most architecture and interior design clients do). The result: incomplete preference data, multiple follow-up rounds, and a first impression that didn't match V+D's premium design positioning.

---

## Solution



![n8n workflow — visual intake form → GPT preference summarisation → auto-generated client brief](images/automation-n8n-workflow.png)
*n8n workflow — visual intake form → GPT preference summarisation → auto-generated client brief*A Typeform-based visual survey with image grids and mood board-style preference selection — connected to a Make.com automation that:
- Collected client aesthetic preferences through visual selection (image-based rather than text)
- Organised responses into structured client preference profiles
- Generated a design brief summary from collected preferences
- Stored all data in OneDrive and triggered the next stage of the engagement workflow

The system was designed around the RIBA Work Plan stages — ensuring the data collected mapped directly to what V+D needed at each project phase.
